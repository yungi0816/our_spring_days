# 기능 명세서

## FEAT-001. 미션 관리

### 설명
사용자는 미션을 만들고, 제한시간과 사진 인증을 통해 완료할 수 있다.

### 입력값

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| content | String | Y | 미션 내용 |
| deadline | DateTime | N | 제한시간 |
| originalImageUrl | String | N | 참고 사진 URL |
| creatorId | String | Y | 작성자 식별값 |

### 처리 로직
1. 입력값 검증
2. 사진이 있으면 이미지 업로드
3. Firestore `missions` 문서 생성
4. 완료 시 인증 사진 URL과 완료 시간을 업데이트

### 예외 처리

| 상황 | 처리 |
|---|---|
| 내용 없음 | 저장하지 않음 |
| 제한시간 오류 | 안내 메시지 표시 |
| 업로드 실패 | Firestore 저장 중단 |

## FEAT-002. 지도 마커 관리

### 설명
사용자는 직접 저장한 장소를 하트 마커로 관리한다.

### 입력값

| 항목 | 타입 | 필수 | 설명 |
|---|---|---|---|
| name | String | Y | 마커명 |
| category | String | Y | 카테고리 |
| address | String | N | 주소 |
| imageUrl | String | N | 장소 사진 URL |
| latitude | double | Y | 위도 |
| longitude | double | Y | 경도 |

### 처리 로직
1. 지도 길게 누르기로 위치 선택
2. 장소 정보 입력
3. 작성자에 따라 마커 색상 결정
4. 저장된 마커 터치 시 상세 패널 표시
5. 수정/삭제 요청을 Firestore에 반영

## FEAT-003. 여행 루트 기록

### 설명
GPS 위치를 주기적으로 기록해 여행 루트를 저장한다.

### 처리 로직
1. 위치 권한 확인
2. 시작 위치 기록
3. 이동거리 필터를 적용해 포인트 추가
4. 자동저장으로 중간 상태 보존
5. 중지 시 마지막 위치를 추가 기록
6. 최종 route 문서 저장

### 안정성 처리
- 자동저장 중 최종저장이 들어오면 짧게 대기한다.
- 대기 시간이 길어지면 최종저장이 우선 진행된다.
- 완료된 루트를 자동저장이 다시 기록중 상태로 덮어쓰지 않는다.

## FEAT-004. 앨범

### 설명
사용자는 여러 장의 사진을 한 번에 선택해 앨범 항목으로 저장할 수 있다.

### 처리 로직
1. 사진 다중 선택
2. 제목과 장소 정보 입력
3. 각 사진을 업로드
4. 사진별로 `albums` 문서 생성
5. 앨범 탭에서 일반 앨범과 미션 사진을 분리 표시

## FEAT-005. 위치 공유

### 설명
앱 사용자의 현재 위치를 Firestore에 저장하고 지도 위에 표시한다.

### 처리 로직
1. 지도 진입 시 위치 권한 확인
2. 현재 위치 저장
3. 일정 간격으로 위치 갱신
4. 파트너별 아이콘 마커로 표시
