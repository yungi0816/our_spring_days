# 트러블슈팅

## 문제 1. 사진 업로드 시 Firebase Storage `object-not-found`

### 상황
프로필 사진 변경, 미션 사진 첨부, 앨범 생성에서 업로드 후 URL 조회가 실패했다.

### 원인
Storage 업로드 직후 객체 조회가 안정적이지 않았고, Storage 설정과 업로드 경로가 화면마다 분산되어 있었다.

### 해결
- 공통 업로드 함수를 하나로 모았다.
- 이후 Cloudinary 업로드 방식으로 전환해 이미지 URL을 직접 받아 저장했다.
- 업로드 설정은 공개 코드가 아니라 빌드 시 주입하도록 분리했다.

### 배운 점
이미지 업로드는 화면마다 따로 구현하면 같은 오류가 반복된다. 앱에서는 업로드 계층을 하나로 두는 편이 낫다.

## 문제 2. 여행기록 중지 후 계속 `저장중`

### 상황
기록 중지를 눌러도 버튼이 비활성화되고 저장중 상태가 유지됐다.

### 원인
자동저장과 최종저장이 같은 `_isWriting` 플래그를 사용했고, 최종저장이 자동저장을 기다리는 동안 상태가 풀리지 않는 흐름이 있었다.

### 해결
- 최종저장 대기 시간을 제한했다.
- 일정 시간이 지나면 최종저장이 우선 진행되도록 했다.
- 완료된 루트를 자동저장이 다시 덮어쓰지 않도록 Firestore transaction을 사용했다.

## 문제 3. 여행 루트가 0m로 저장됨

### 상황
루트 목록에는 기록이 생기지만 거리가 0m이고 지도에는 시작 지점만 표시됐다.

### 원인
이동거리 필터가 짧은 이동이나 마지막 위치를 버릴 수 있었다. 중지 시점에 최종 위치를 추가로 기록하지 않았다.

### 해결
- 중지 시 현재 위치 또는 마지막 known 위치를 한 번 더 기록한다.
- 강제 기록 시 작은 이동거리 필터를 우회한다.

## 문제 4. 지도 안내 패널이 마커 정보를 가림

### 상황
상단 안내 패널이 마커 info window를 가려 장소 정보를 보기 어려웠다.

### 해결
- 안내 패널에 닫기 버튼을 추가했다.
- 닫은 뒤에는 우측 상단 작은 아이콘으로 다시 열 수 있게 했다.
- 저장 마커는 별도 상세 바텀시트에서 수정/삭제한다.

## 문제 5. 갤럭시 하단 내비게이션 버튼에 UI가 가림

### 상황
하단 탭과 일부 버튼이 시스템 내비게이션 영역과 겹쳤다.

### 해결
하단 네비게이션을 `SafeArea(top: false)` 안에 넣어 기기별 하단 inset을 반영했다.
