# 요구사항 정의서

## 기능 요구사항

| ID | 기능명 | 설명 | 우선순위 |
|---|---|---|---|
| REQ-001 | 사용자 전환 | 두 명의 사용자를 전환하며 앱을 사용할 수 있다. | 상 |
| REQ-002 | 미션 등록 | 사용자는 텍스트, 제한시간, 참고 사진을 포함해 미션을 등록할 수 있다. | 상 |
| REQ-003 | 미션 인증 | 진행 중인 미션에 인증 사진을 첨부해 완료 처리할 수 있다. | 상 |
| REQ-004 | 미션 수정 | 작성자는 미션 내용과 제한시간을 수정할 수 있다. | 중 |
| REQ-005 | 지도 장소 저장 | 지도 길게 누르기로 장소명, 주소, 카테고리, 사진을 저장할 수 있다. | 상 |
| REQ-006 | 지도 장소 수정/삭제 | 저장된 마커를 다시 눌러 정보를 수정하거나 삭제할 수 있다. | 상 |
| REQ-007 | 추천 장소 검색 | 지도 주변 애견 동반 맛집·카페를 검색할 수 있다. | 상 |
| REQ-008 | 외부 지도 연동 | 추천 장소 마커를 누르면 네이버지도 앱 또는 웹으로 연결된다. | 중 |
| REQ-009 | 위치 공유 | 현재 사용자 위치를 저장하고 지도에 파트너별 아이콘으로 표시한다. | 상 |
| REQ-010 | 여행 루트 기록 | GPS를 사용해 이동 경로, 거리, 시간을 기록한다. | 상 |
| REQ-011 | 루트 상세 조회 | 저장된 여행 기록을 전체/일자별로 지도에서 확인한다. | 상 |
| REQ-012 | 앨범 생성 | 여러 장의 사진을 한 번에 업로드해 앨범으로 저장한다. | 중 |
| REQ-013 | 다국어 지원 | 한국어와 일본어 UI를 전환할 수 있다. | 중 |

## 비기능 요구사항

| ID | 항목 | 설명 |
|---|---|---|
| NFR-001 | 개인정보 | API 키, 실제 사진, 개인 위치 기록은 공개 저장소에 포함하지 않는다. |
| NFR-002 | 성능 | 지도 마커와 루트 표시는 일반적인 모바일 환경에서 즉시 조작 가능한 수준을 유지한다. |
| NFR-003 | 안정성 | 여행기록 중지 시 저장 상태가 고착되지 않아야 한다. |
| NFR-004 | 접근성 | 주요 버튼은 아이콘과 텍스트를 함께 제공한다. |
| NFR-005 | 호환성 | Android 중심으로 구현하고 iOS 대응 문서를 별도로 남긴다. |
| NFR-006 | 유지보수 | 기능별 화면과 Provider를 분리해 변경 범위를 좁힌다. |

## 공개 저장소 요구사항

| ID | 항목 | 설명 |
|---|---|---|
| OSS-001 | Secret 분리 | Google Maps, Firebase, Cloudinary 설정은 빌드 시 주입한다. |
| OSS-002 | 문서화 | 요구사항, 구조, 테스트, 배포, 트러블슈팅 문서를 포함한다. |
| OSS-003 | 빌드 산출물 제외 | APK, AAB, 로컬 설정 파일은 Git에 포함하지 않는다. |
