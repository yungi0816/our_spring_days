# 사용자 시나리오

## 시나리오 1. 미션을 만들고 인증한다

### 사용자
파트너 A, 파트너 B

### 흐름
1. 파트너 A가 메인 화면에서 `미션시작`을 누른다.
2. 미션 내용을 입력한다.
3. 제한시간을 선택하거나 직접 입력한다.
4. 필요하면 참고 사진을 첨부한다.
5. 저장하면 진행 중인 미션 목록에 표시된다.
6. 파트너 B가 미션 사진을 첨부한다.
7. 미션이 완료 처리되고 인증 사진이 앨범 탭에 표시된다.

### 예외 상황
- 제한시간 직접 입력값이 숫자가 아니면 저장하지 않는다.
- 사진 업로드 설정이 없으면 오류 메시지를 표시한다.
- 작성자가 아닌 사용자는 미션 수정/삭제 버튼을 보지 않는다.

## 시나리오 2. 지도에 데이트 장소를 저장한다

### 흐름
1. 사용자가 지도 탭을 연다.
2. 지도를 길게 눌러 장소 추가 창을 연다.
3. 마커명, 주소, 카테고리, 사진을 입력한다.
4. 저장된 장소는 작성자에 따라 빨간색/파란색 하트 마커로 표시된다.
5. 저장된 마커를 다시 누르면 상세 패널이 열린다.
6. 사용자는 마커 정보를 수정하거나 삭제한다.

### 예외 상황
- 장소명이 비어 있으면 저장하지 않는다.
- 사진 업로드 실패 시 저장을 중단하고 오류를 표시한다.

## 시나리오 3. 애견 동반 장소를 찾고 네이버지도로 연다

### 흐름
1. 지도 상단에서 `애견 맛집` 또는 `애견 카페`를 누른다.
2. Google Places API로 주변 장소를 검색한다.
3. 추천 장소는 기본 Google 마커로 표시된다.
4. 추천 마커를 누르면 네이버지도 앱으로 장소를 연다.
5. 네이버지도 앱이 없으면 웹 검색 또는 스토어로 이동한다.

## 시나리오 4. 여행 루트를 기록한다

### 흐름
1. 메인 화면에서 `여행기록`을 누른다.
2. 앱이 위치 권한을 확인한다.
3. 기록 중에는 시간과 이동거리가 표시된다.
4. 사용자가 `기록 중지`를 누른다.
5. 마지막 위치를 한 번 더 기록한 뒤 Firestore에 저장한다.
6. 루트 탭에서 이동 경로, 거리, 시간, 방문 후보 장소를 확인한다.

### 예외 상황
- 위치 권한이 없으면 기록을 시작하지 않는다.
- 최종 저장 중 자동저장과 충돌해도 저장 상태가 계속 고착되지 않는다.
