# 프로젝트 회고

## 잘한 점

- 기능을 화면 단위로 나누고 Riverpod Provider로 상태를 분리했다.
- 미션, 지도, 앨범, 루트가 서로 연결되는 흐름을 만들었다.
- 강아지와 여행하는 커플이라는 사용 장면을 기준으로 애견 동반 장소 검색과 루트 기록을 묶었다.
- 사진 업로드와 여행기록처럼 오류가 반복되는 부분을 공통 로직으로 정리했다.
- 실제 기기에서 드러나는 하단 안전영역, 위치 권한, 저장중 고착 문제를 반영했다.

## 어려웠던 점

- GPS 기록은 에뮬레이터만으로 확인하기 어렵고, 실제 이동 환경에서만 보이는 문제가 있었다.
- 지도 마커는 추천 장소와 직접 저장한 장소의 역할이 달라서 표시 규칙을 분리해야 했다.
- 공개 저장소 준비 과정에서 API Key와 로컬 설정을 코드에서 분리하는 작업이 필요했다.

## 개선할 점

- Firebase Auth를 붙여 실제 계정 기반으로 확장한다.
- 루트 기록을 백그라운드 환경에서 더 오래 안정적으로 테스트한다.
- Firestore Rules 예시와 보안 테스트 문서를 추가한다.
- GitHub Actions로 `flutter analyze`와 `flutter test`를 자동 실행한다.

## 배운 점

앱은 기능이 많아지는 순간 저장 흐름과 예외 처리가 더 중요해진다. 특히 위치 기록, 이미지 업로드, 외부 앱 연동은 단순 구현보다 실패했을 때의 복구 흐름을 먼저 생각해야 한다.
