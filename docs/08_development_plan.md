# 개발 계획서

## 개발 단계

| 단계 | 작업 내용 | 산출물 |
|---|---|---|
| 1단계 | 앱 컨셉 정리, 주요 화면 정의 | 프로젝트 개요 |
| 2단계 | 상태관리, Firestore 모델 설계 | Provider, 모델 클래스 |
| 3단계 | 미션 기능 구현 | 미션 등록/인증/수정 |
| 4단계 | 지도 기능 구현 | 장소 저장, 추천 검색, 외부 지도 연동 |
| 5단계 | 여행기록 구현 | GPS 기록, 루트 상세 화면 |
| 6단계 | 앨범 기능 구현 | 다중 업로드, 미션 사진 표시 |
| 7단계 | 안정화 | 업로드 오류, 저장 고착, 안전영역 수정 |
| 8단계 | 공개 저장소 정리 | 문서, 시크릿 분리, README |

## 작업 관리 방식

- 기능 단위로 화면과 Provider를 나눠 수정한다.
- 저장 로직은 Firestore 모델 변경 범위를 먼저 확인한다.
- 지도/위치 기능은 실제 기기 테스트를 우선한다.
- 공개 전에는 API Key와 빌드 산출물을 제외한다.
