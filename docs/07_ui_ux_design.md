# UI / UX 설계서

## 화면 목록

| 화면 ID | 화면명 | 설명 |
|---|---|---|
| UI-001 | 스플래시 | 앱 진입 화면 |
| UI-002 | 메인 | 미션 시작, 여행기록 시작/중지, 진행 중 미션 |
| UI-003 | 지도 | 장소 저장, 추천 장소 검색, 위치 공유 |
| UI-004 | 미션 | 미션 이력 및 상태 요약 |
| UI-005 | 앨범 | 앨범 사진과 미션 인증 사진 확인 |
| UI-006 | 루트 | 여행 기록 목록과 상세 지도 |

## 메인 화면

### 구성 요소
- 사용자 프로필 버튼
- 언어 전환 버튼
- 미션시작 네모 버튼
- 여행기록 네모 버튼
- 여행 기록 상태 영역
- 진행 중 미션 목록

### UX 포인트
- 두 핵심 행동을 첫 화면 상단에 배치한다.
- 기록 중에는 시간과 거리를 계속 보여준다.
- 저장 중 상태가 길어지지 않도록 최종 저장 흐름을 분리한다.

## 지도 화면

### 구성 요소
- Google Map
- 직접 저장한 하트 마커
- 추천 장소 기본 마커
- 현재 위치 공유 마커
- 애견 맛집/카페 검색 버튼
- 장소 검색 버튼
- 현재 위치 이동 버튼
- 접고 펼칠 수 있는 안내 패널

### 마커 규칙

| 마커 | 표시 |
|---|---|
| 파트너 A 저장 장소 | 빨간 하트 |
| 파트너 B 저장 장소 | 파란 하트 |
| 추천 장소 | 기본 Google 마커 |
| 현재 위치 | 사용자 캐릭터 아이콘 |

## 앨범 화면

### 구성 요소
- 앨범 탭
- 미션 탭
- 사진 그리드
- 상세 보기 바텀시트
- 저장 버튼

### UX 포인트
- 여러 장 선택 시 가로 썸네일로 확인한다.
- 일반 앨범과 미션 인증 사진을 분리한다.
- 저장 방식은 일반 저장과 테마 저장으로 나눈다.

## 하단 안전영역

Galaxy처럼 시스템 내비게이션 버튼이 있는 기기에서 버튼이 가려지지 않도록 하단 네비게이션을 `SafeArea` 안에 배치한다.
